# Sprint 01 - Base Navegavel do Produto

## Objetivo

Transformar o prototipo validado em uma primeira aplicacao navegavel do produto final, ainda sem banco de dados, mas com estrutura de experiencia, regras comerciais, operacao MB e dados simulados consistentes.

## Entregas ja iniciadas

- pasta separada do produto final
- prototipo atual preservado como referencia
- aplicacao web inicial em `apps/web`
- login unico visual
- portal do cliente
- administracao MB
- planos com valores editaveis
- persistencia local simulada para valores e plano do cliente
- inteligencia financeira unificada
- DRE e fluxo de caixa no plano CFO
- relatorios com impressao e exportacao CSV/Excel
- area MB para alimentar dados do cliente

## Proximas tarefas

1. Refinar visual do cockpit principal.
2. Criar tela de onboarding do cliente.
3. Criar tela de importacao de arquivos Excel, CSV, OFX e XML.
4. Melhorar comunicacao operacional cliente x MB.
5. Criar historico de auditoria visual no admin.
6. Criar tela de usuarios e perfis.
7. Criar cadastro visual de novo cliente pela equipe MB.
8. Criar fluxo visual de compra do plano em etapas.
9. Criar modelos de relatorios padronizados mais proximos do layout final da MB.
10. Preparar checklist para migrar de dados simulados para banco de dados.

## Criterios de aceite

- a aplicacao abre direto pelo navegador
- cliente consegue navegar sem entender tecnologia
- equipe MB consegue entender como operaria clientes reais
- diferenca entre planos fica evidente
- valores de planos podem ser alterados no admin
- cliente CFO visualiza DRE, caixa, graficos e pareceres
- documentos deixam claro que envio principal e da equipe MB
- sistema transmite produto SaaS premium, nao ERP tradicional

## Riscos controlados

- ainda nao ha seguranca real
- ainda nao ha banco de dados
- ainda nao ha integracao real
- ainda nao ha pagamento real
- ainda nao ha IA real

Esses pontos ficam fora da Sprint 01 para preservar foco e velocidade.
