1. Sumário executivo

O MB Intelligence é um portal de entrega + inteligência da contabilidade — não um ERP financeiro transacional. A MB valida e publica dados (DRE, fluxo, score, documentos, guias) e o cliente consulta pelo portal. Funciona em produção; o esqueleto é sólido. O maior débito não é técnico, é de valor percebido (ainda mostra mais "dados" que "inteligência") e de higiene (código legado/protótipo convivendo com o de produção).

2. Arquitetura e stack

NavegadorSPA em JavaScript puro (sem framework). Renderiza via innerHTML, namespace global window.MBI. localStorage como espelho offline.
→
VercelEstático (apps/web) + 1 função serverless api/handler.js que carrega server-supabase.js.
→
Backend Nodeserver-supabase.js (1.385 linhas) — roteamento, regra de negócio e acesso a dados em um arquivo. Valida token contra o Supabase.
→
SupabasePostgreSQL + Auth + Storage. 15 tabelas, RLS habilitada. Backend acessa via service_role.

Onde NÃO há rotina automática: a importação só registra metadados (não processa o arquivo); não existe contas a pagar/receber; não há alertas automáticos nem e-mail/WhatsApp ao publicar. Tudo depende da MB inserir manualmente. É aqui que está a maior oportunidade (a "Inteligência MB") e a maior fragilidade operacional.

Conclusão

O sistema funciona e tem uma base sólida. Para virar um produto que vende, a prioridade não é mais "fazer funcionar" — é (1) entregar inteligência (diagnóstico + alertas com a marca MB), (2) limpar o legado e (3) fechar os bloqueadores de produção (staging, testes, backup, contato real). Posicionar a MB como a inteligência contábil do empresário — não como mais um ERP.
